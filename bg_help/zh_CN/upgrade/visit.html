访问方式设置

 
填写说明

默认

此方式为 PHP 最常用的访问方式，使用查询串的方式访问网站，URL 的形式为 http://www.domain.com/index.php?m=article&a=show&article_id=123；优点是效率较高，兼容性好；缺点是 URL 友好度较差，对于搜索引擎的收录有一定影响。

伪静态

此方式目前较为流行，使用友好的 URL 方式访问网站，URL 的形式为 http://www.domain.com/article/123，此方式需要服务器支持 URL 重写 (URL Rewriting)，具体请查看 互动百科；优点是 URL 较为友好，有利于搜索引擎的收录；缺点是效率相对较低，兼容性差，且需要重新配置服务器；在 Apache 环境下，如支持 URL 重写，安装程序会自动帮您设置，是否支持 URL 重写，请咨询服务器提供商。Nginx 环境下，需下载相关的配置文件，重新配置服务器，点此下载配置文件；IIS 环境暂不支持伪静态模式。

纯静态

根据不同系统，有可能无此选项。此方式为效率最高的访问方式，利用静态页面生成模块，将所有栏目、文章等内容生成为静态 HTML 文件，URL 的形式为 http://www.domain.com/article/123.html，此方式需开启静态页面生成模块（开启方法请查看 常见问题）；此方式的优点是效率最高，无数据库服务器压力，兼容性好，有利于搜索引擎的收录；缺点是占用硬盘空间相对较多，页面需要定时生成。

生成静态页面类型

根据不同系统，有可能无此选项。此选项决定生成静态页面的类型。
